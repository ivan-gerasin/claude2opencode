# claude2opencode

A small command-line tool that converts Claude Code skills, slash commands, and
subagents (including ones bundled in installed plugins) into OpenCode-compatible
`SKILL.md` files.

## What it does

OpenCode and Claude Code both use the same `SKILL.md` format (YAML frontmatter +
Markdown body), but:

- Claude Code skills/plugins live scattered across `~/.claude/...`, which
  OpenCode doesn't scan by default ‚Äî they need to be gathered into
  `~/.config/opencode/skills/`.
- OpenCode validates frontmatter more strictly (only `name`, `description`,
  `license`, `compatibility`, `metadata` are recognized; `name` must match
  `^[a-z0-9]+(-[a-z0-9]+)*$` and match its directory name; `description` must be
  1‚Äì1024 characters).
- Claude plugins also bundle slash commands and subagents, which have no
  OpenCode "skill" equivalent ‚Äî this tool wraps them into synthetic skills
  (`cmd-<name>` / `agent-<name>`) so their guidance stays available.

The script discovers everything under `~/.claude/skills/` plus the
skills/commands/agents bundled in every currently-installed plugin, normalizes
each into a valid OpenCode `SKILL.md`, and reports what it did (converted,
skipped, warnings) at the end.

See `docs/2026-06-08-claude-to-opencode-skill-converter-design.md` for the full
design rationale and conversion rules.

## Usage

```
python3 convert_skills.py [--claude-dir DIR] [--output-dir DIR] [--dry-run]
```

- `--claude-dir` ‚Äî Claude Code config directory (default: `~/.claude`)
- `--output-dir` ‚Äî OpenCode skills directory to write to (default:
  `~/.config/opencode/skills`)
- `--dry-run` ‚Äî preview the conversion and print the report without writing
  any files

Run with `--dry-run` first and read through the report before doing a real run ‚Äî
existing directories with colliding names in the output directory are
overwritten.
